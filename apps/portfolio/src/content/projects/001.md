JSP 기반 레거시를 Next.js로 전환한 프로젝트. 프론트엔드 2인이 5개월 안에 3개 서비스 100여 개 페이지를 완성했습니다.

## 담당 파트
- 인증 아키텍처 설계 및 구현 (x-auth 헤더 기반)
- 공통 컴포넌트 & 공통 훅 개발
- 콘텐츠, 이벤트 등 주요 도메인 페이지 구현
- Next.js 16 마이그레이션

## 핵심 도전
**일정 압박 속 레거시 전환**: 바로 구현 대신 피그잼으로 페이지 구조를 먼저 분석하고 공통 컴포넌트와 훅을 선구축했다. axios 인터셉터와 QueryCache 전역 핸들러로 에러 처리를 공통화해 이후 페이지 개발 속도를 높였다.

**서버-클라이언트 인증 불일치**: 세션 기반 인증에서 서버 컴포넌트는 쿠키에 접근 가능하지만 클라이언트는 불가해 인증 상태가 어긋났다. Middleware에서 세션을 선행 검증 후 암호화된 유저 정보를 x-authorization 헤더에 주입하고, AuthProvider로 클라이언트에 즉시 동기화했다. 트레이드오프로 전 페이지 SSR이 강제되었고, filter 레이어로 인증 불필요 경로를 분기해 부분 완화했다.

## 배운 것
급할수록 공통화에 먼저 투자해야 한다. 공통 훅과 에러 처리 공통화가 전체 개발 속도를 높였다.
