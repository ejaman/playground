교육과정 변경에 따라 콘텐츠가 추가될 때마다 개발 공수 없이 JSON 데이터만으로 신규 콘텐츠를 확장할 수 있는 렌더링 아키텍처 1인 설계. 빌드 시 콘텐츠별로 독립적인 JS/HTML 산출물이 생성됩니다.

## 담당 파트
- JSON 스키마 설계 (콘텐츠 구조 정의 및 타입 시스템)
- 데이터 주도형 렌더링 아키텍처 구현
- 콘텐츠별 독립 빌드 파이프라인 구축 (Vite rollupOption)
- 콘텐츠 타입별 컴포넌트 구현 (영상, 학습지, 활동, 퀴즈 등)

## 핵심 도전
**JSON → TypeScript 스키마 전환**: 초기 JSON으로 작성하면 오타와 잘못된 타입이 런타임까지 통과되어 에러가 발생했다. ts 파일로 작성 후 JSON으로 변환하는 방식으로 개선해 컴파일 타임 타입 검증을 확보했다. Tagged Union을 적용해 콘텐츠 타입마다 자동으로 타입이 좁혀지는 구조를 설계했다.

**다중 콘텐츠 빌드 파이프라인**: 소스는 하나의 레포지만 콘텐츠별 독립 실행 파일이 필요했다. Node.js fs로 각 콘텐츠 폴더의 진입점을 자동 감지 후 rollupOption에 동적으로 주입해 단 한 번의 빌드 명령으로 모든 콘텐츠를 병렬 처리했다.

## 배운 것
타입은 단순 에러 방지 수단이 아닌 설계다. 타입 구조만 잘 잡아도 컴포넌트가 어떤 데이터를 받고 어떻게 동작해야 하는지 자연스럽게 정의된다.
